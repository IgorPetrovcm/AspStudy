HTTP - распространненый протокол передачи данных, изначально предназваченный для передачи гипертекстовых документов. Так и расшифруется название 'HyperText Transfer Protocol', то есть 'Протокол передачи гипертекста'

Задача решаемая с помощью HTTP протокола - обмен данными между пользовательским прилодением, осуществляющим доступ к веб-ресурсам(веб-браузер итд) и веб-сервером.

API многих программ подразумевает использование HTTP для передачи данных - сами данные могут иметь любой формат, например XML, JSON итд.

Если пользователь ввел в командной строке браузера допустим:
http://alizar.habrahabr.ru/
То браузер подключается к веб-серверу по этому адресу

Для такого подключения мы можем воспользоваться утилитой telnet:
telnet alizar.habrahabr.ru 80(это порт, который используется по умолчанию)

После того как вы подключились к серверу, можно отправить HTTP-запрос
Для того, чтобы сформировать HTTP-запрос, необходимо составить стартовую строку, а также задать как минимум один заголовок - это заголовок Host, который является обязательным.
Дело в том, что преобразование доменного имени в IP-адрес осуществляется на стороне клиента, и когда вы открываете TCP-соеденение, то сервер не знает какой именно адрес использовался для соеденения.
Фактически сетевое соеденения открываться с одним и тем же узлом, но деменные имена разные и сервер этого не знает - именно пожтому этот IP-адрес необходимо передать в заголовке Host.

Схема запроса:
Метод URI HTTP/Версия
Например:
GET / HTTP/1.1

Метод определяет операцию, которую нужно осуществить с указанным ресурсом. HTTP1.1 не ограничивает количество использованных методов.

URI(Uniform Resourse Identifier)(Уникальный тндетификатор ресурса) - путь до конкретного ресурсаб над которым необходимо осуществить операцию. Некоторые запросы могут не относить к каму либо конкретному ресурсу, в этом случае вместо URI вставляют '*' звездочку.
OPTIONS * HTTP/1.1

Обратимя к веб-странице по определенному адресу:
GET / HTTP/1.1
Host: alizar.habrahabr.ru

Стартовая строка ответа имеет следующую структуру:
HTTP/Версия КодСостояния Пояснение

Версия - протокол
Код состояния - три цифры (первая из которых указывает на класс состояния), которые определяют результат совершения запроса. Например при использовании метода GET, сервер предоставляет ресурс с указанным идентификатором, то такое состояние задаетсмя кодом 200. Если сервер сообщает о том, что такого ресурса не существует - 404. Если сервер не предоставляет ресурс по причине отсутствия тех или иных привелегий - 403.
